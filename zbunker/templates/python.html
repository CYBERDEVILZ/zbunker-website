Python OOP: Classes & Objects

This video marks the beginning of the Object Oriented Programming (OOP) side of Python. OOP is a programming model where we make use of classes and objects to represent our codes. It is considered one of the neatest and most logical form of programming. Class is an encapsulation of similar data (called attributes) and functions (called methods). Objects can access these attributes and methods. Each object that you create would have its own blueprint of the class which it is representing.

Python OOP: Self in Detail

Self word might be pretty confusing for beginners who enter the realm of OOP in Python. But, believe me, it is pretty straight-forward and easy to understand concept. Work and experiment with self, and you would master it in no time! Self is not a keyword by the way, it is a convention. You can name it anything you want, but developer across the world know it as self and I too recommend you to follow the same convention.

Python OOP: Class Variables

String Interpolation

Every beginner level programmers make this sort of mistake when it comes to plugging in variables in a string. They concatenate the variable with the string in order to achieve this. But, this is in general, a very wrong practice and prone to error. To overcome this, we make use of String Interpolation. SI is nothing but making use of placeholders to fill in the variable value in the string. This short video will get you started with this concept and make you understand really well.

First Class Function

Everything in Python is a proper object. This includes Functions too. Now, many of us haven't heard about this earlier, but this is actually true. Functions, like any other object, can be passed as arguments, be renamed etc. This video will extensively cover the fundamentals of functions being a first class object and how FCF can pave the way or open the doors for other developmental practices.

Closures

Python functions acts just like other ordinary first class objects. This property formed the base of various other interesting 'phenomena' like closures. Closure is when a function object is able to remember the value in its enclosing scope. Nesting a function is the condition for closures to happen. In this video, we will cover closure in detail by taking a couple of examples.

Ever found a python code that makes use of '@' operator above any function? Well, upon doing a quick search you will find out that these types are called decorators. Now, decorators can be either functions or classes. But most prominent form being functions. In this video, we will be looking at decorators (function) in a simple and a lucid manner. Decorators basically extends the functionality of a function without explicitly modifying it. Don't worry if this doesn't make sense to you. We will cover more about it in this video.
