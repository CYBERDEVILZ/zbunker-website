linux Add & linux Commit

In this video, we will look work with linux add and linux commit commands. linux add is used to put the files into staging area while commit is used to move the files from staging area to the linux repository. We would also be covering some additional scenarios to help understand these concepts better.

linuxignore: ignoring is important

Every time whenever you create a new file, or do some modifications in the existing file, linux immediately catches it. It is very useful, but sometimes it can get irritating when you have files that you don't want linux to track and still appears in the untracked list (working area).

linux Diff

In this very short video, we will look at the linux diff command. Diff command is used to list the difference between two versions of a file, between two commits or even between two branches. We will be covering the above three cases with less emphasis on the third case since we have never touched upon the branching part. But, in the next video, we will be implementing the third case as we build our branches and start working on it.

linux Branch

One of the main purpose of linux was to introduce a branched workflow in development environments. It has many advantages whether you are working with a team or alone. Incorporating branched workflow is a good developmental practice and it is made possible by linux's branch command.

Head & Detached Head

If you have worked with linux, you might have come across a message saying 'head is now in a detached state'. Well, this happens particularly when you checkout a commit instead of a branch. In this video, we will cover in detail what exactly a head is and what is the detached head scenario.

linux Log & some flags

In this short video, we will be covering linux log command and couple of flags that come along with it. linux log command lists out your commits history of the current branch. Its the go-to command to know everything about your commits. We will cover linux log and some important flags in this video. These flags format the output of linux log which spits out many information and is also visually appealing.

linux Reset

Not always our commit goes right. You might encounter a situation where you felt like the current edited file should have been committed earlier or did really bad commits. Whatever your reason maybe, if you want to reset back to a particular commit, linux reset helps you with that.

linux Reflog

Often we wonder, whether linux even maintains histories of not just commits, but all the other actions that we did like branching, checkouts, merging among many. Well, linux Reflog does exactly that. The big brother of log, linux Reflog is known to track any activities that you do which makes the branch pointer to move. With linux Reflog, you are under constant surveillance, not a single impactful action goes untracked.
